Boggle Solver

backtracking, dfs

This project is a 4x4 Boggle solver using backtracking. The inspiration for this project came from my family, and although we play a lot of Boggle, we're never quite sure how well we really did. Because of this, I decided to code an algorithm that would give me the solutions to any board. After thinking for some time, I decided on a backtracking algorithm to search for solutions efficiently and store them in a space-optimized manner.

Originally, the code for this project was written in C++ using many of the STL's data structures, but I adapted it to C# to interface with this website, but my free Azure account doesn't allow for so much computing. However, by leveraging the power of ASP.NET Core and its ease of use with forms, I should be able to add in the web component soon. Additional things to add to this application on the web side are more comprehensive form validation, logging a "high score" board, and adding functionality to automattically tab over to the next input space.

Make sure to check out this project's GitHub to get in-depth coverage of the algorithm I implemented and the data structures I used to accomplish this!

Event Scheduler

backtracking, dfs, big-o

My inspiration for this project came from talking with my girlfriend and her mom at a horse riding competition. In the world of Eventing, technology is relatively new and I saw a way I could put some of my knowledge on backtracking algorithms to use. At these Events, there are multiple riders riding in several divisions, many of which have the same coach. Each rider has three ride times throughout multiple days (one for each Event) and there are often conflicts between riders who have the same coach. Because of these time-based conflicts, I immediately had the idea to solve this problem with backtracking.

Now if I lost you in the paragraph above, don't worry, to simplify, this is a C++ project to solve time-based conflicts between coaches in a complex sporting event that has multiple start times and end times.

This project is presented as a solution to this time-based conflict. The user inputs all relevant information and is presented with a non-conflicting schedule for all of the riders in any competition. Check out the algorithmic details on GitHub!

UPS 2020 Hackathon

web-app, hackathon-project

The Career Constructor is a quiz-making web tool for recruiters and talent acquisition specialists to screen applicants based on relevant knowledge. It allows users to create questions, group these questions into quizzes, and assign and grade these quizzes to and from applicants. The value of this project is two-fold. One, it allows HR specialists and managers an easy way to ensure their applicants have relevant knowledge, and two, it highlights candidates with this knowledge in the interview process. The application is aimed towards interns who don't have much real-world experience but have the technical skills. By taking these quizzes they show off what they know, essentially constructing their careers.

The project was built using ASP.NET Core, which inherently pulled on C#, HTML/CSS, etc.. We used a local SQL database for storing questions, quizzes, and users, but our group had intentions to use cloud services and extend this application much further than we were able to in just a short two day time period.
